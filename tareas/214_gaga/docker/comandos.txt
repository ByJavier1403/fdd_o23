Docker Command Cheatsheet:

1) docker pull <docker_image>: Downloads images to your local machine.

2) docker run <options> <docker_image>: Runs a container of the specified Docker image.

   2.1) docker run -d: Run in detached mode.

   2.2) docker run <image:tag>: Combines docker pull and docker start in a single command.

   2.3) docker run -p <local_port>:<container_port>: Runs the container and binds a port on your local machine to the container's port.

6) docker stop <container_id>: Stops a specified container.

7) docker start <container_id>: Starts an existing container.

8) docker ps: Displays a list of all currently running Docker containers.

9) docker ps -a: Lists all containers, both running and stopped.

10) docker images: Lists all available Docker images on your local machine.

11) docker exec -it <container_id> /bin/bash: Allows interactive terminal access within a Docker container. Provide the container ID followed by /bin/bash.

12) docker logs <container_id>: Shows logs generated by a running container, providing insights into the application's behavior. 

13) docker run --name <new_container_name> <image>: Runs a container and assigns a custom name.

14) docker rm <container_id|container_name>: Deletes a stopped container without removing the associated image.

15) docker rmi <image_id>: Deletes a Docker image.

16) docker build -t <image_name:tag> <path>: Builds a new Docker image using a Dockerfile located at the specified path.
